StrategyAI — AWS Architecture

Enterprise AWS · EKS on EC2 node groups · Okta + Azure AD · Amazon Bedrock (LLM) · PrivateLink to vendor MSTR
End-to-end flow: User loads React via CloudFront → logs in with Okta (JWT with Azure AD claims) → API call routed through WAF → API Gateway (validates JWT) → Internal ALB → EKS (FastAPI pods: RBAC + intent parsing via Amazon Bedrock, which also generates embeddings for pgvector RAG in RDS). Long-running MSTR ops (cube refresh, report exec) are enqueued on SQS → Worker pods poll MSTR via AWS PrivateLink into the vendor network. Status written to Redis (live) + RDS PostgreSQL (audit with real user identity). Secrets Manager holds MSTR admin service account, DB password, and Okta client secret — read at runtime via IAM Roles for Service Accounts (IRSA). Observability: CloudWatch logs + metrics + alarms.
