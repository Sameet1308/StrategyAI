StrategyAI — End-to-End Flow

One user action "refresh cube ABC" · 7 system lanes · 13 numbered hops in time order

Sequence — "refresh cube ABC"

1User loads the React SPA from CloudFront → S3.
2User logs in via Okta — browser receives JWT with Azure AD group claims.
3User types "refresh cube ABC" → browser POSTs to AWS WAF with the JWT.
4WAF applies OWASP rules and forwards to API Gateway.
5API Gateway validates the Okta JWT via JWKS and forwards to Internal ALB.
6ALB routes to an EKS FastAPI pod; FastAPI checks AD-group RBAC.
7FastAPI sends the chat text to Amazon Bedrock → receives {intent, payload}.
8FastAPI drops the job on SQS, returns {job_id}; a Worker pod dequeues it.
9Worker calls MSTR REST over AWS PrivateLink using the admin service account.
10Worker polls MSTR every few seconds until the operation finishes.
11Worker sends raw MSTR result back to Bedrock (second call) → plain-English summary.
12Worker persists: audit row + chat → RDS; live status + summary → Redis; large files → S3.
13Browser has been polling /jobs/{id} — FastAPI reads Redis → returns summary → UI displays the result.
Cross-cutting (not drawn): Secrets Manager reads via IRSA on every pod · CloudWatch logs + metrics from every AWS service and pod · KMS envelope encryption on RDS / S3 / Secrets Manager · ECR image pulls on pod start. These happen continuously, not at any single step — omitted to keep the main flow readable.
